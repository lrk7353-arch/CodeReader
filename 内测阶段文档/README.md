# CodeReader 内测阶段文档

阶段版本：`0.11.0-beta.3`

阶段起点：2026 年 6 月 12 日

## 阶段定义

MVP 已于 `v0.10.0-mvp` 完成 Windows x64 基线验收。当前进入内测迭代，不再以“证明最小闭环可行”为主要目标，而是验证真实用户持续使用时的可靠性、可理解性、数据兼容性和问题可诊断性。

内测阶段的核心问题是：

> CodeReader 能否在真实项目、真实模型和持续版本升级中稳定保存用户的代码认知资产，并让问题可复现、可定位、可修复？

## 当前承诺

1. 已有 MVP 用户数据库必须可迁移，不以删除本地数据作为常规升级方案。
2. 合入内测主线前必须通过前端测试与构建、Rust 测试、Clippy 和 Cargo check。
3. 解释生成失败不得覆盖已有解释；外部模型调用必须保留明确的用户确认。
4. 内测问题必须记录复现环境、输入范围、期望结果、实际结果和版本。
5. Windows x64 仍是当前正式发行平台；macOS/Linux 属于后续验证目标，不在本阶段虚假承诺。
