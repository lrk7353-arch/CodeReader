# 发布签名与安全门禁

## 当前状态

CodeReader 的 Windows 安装包在拿到可信代码签名证书之前不会进行 Authenticode 签名。`publisher` 配置只影响安装包元数据，不等于可信代码签名，也不会消除 SmartScreen 的未知发布者提示。当前 Beta 2 阶段已经在 `scripts/sign-windows-artifacts.ps1` 中落地签名与验证框架，未配置证书时仍可完成内测打包，并显式记录为 “未签名内部测试版”。

## 分发规则

- 未签名包只能用于开发者本人或已知内测成员，并必须明确提示“未签名内部测试版”。
- 面向不受控用户、公开下载页或自动更新渠道分发前，必须完成安装包签名、时间戳和发布后验证。
- 不以关闭 SmartScreen、指导用户忽略系统告警或使用自签名证书冒充正式发布作为替代方案。

## 正式签名前置条件

1. 确认真实发布主体和与证书一致的 Publisher 名称。
2. 获取受 Windows 信任的代码签名证书，明确普通 OV 或 EV 证书的保管方式。
3. 私钥使用硬件令牌、受控证书库或签名服务；PFX、密码和访问令牌不得提交 Git。
4. CI/发布机只授予发布任务所需的最小权限，并记录证书指纹和签名时间。
5. 使用可信 RFC 3161 时间戳服务，避免证书到期后已有签名立即失效。

## 签名与验证框架

`scripts/sign-windows-artifacts.ps1` 在 `npm run release:windows` 中被自动加载，提供以下能力：

- 通过环境变量启用签名，不强制要求证书即可运行：
  - `CODEREADER_CODESIGN_PFX_PATH` + `CODEREADER_CODESIGN_PASSWORD`：使用 PFX 文件；
  - `CODEREADER_CODESIGN_SHA1`：使用 Windows 证书库中已安装证书的 SHA1 指纹；
  - `CODEREADER_CODESIGN_TIMESTAMP_URL` + `CODEREADER_CODESIGN_TIMESTAMP_ALG`：配置 RFC 3161 时间戳；
  - `CODEREADER_CODESIGN_DIGEST_ALG`：摘要算法，默认 `sha256`；
  - `CODEREADER_CODESIGN_REQUIRED=1`：未配置证书时直接中断发布。
- 在签名时优先使用本机或 Windows SDK 自带的 `signtool.exe`，未找到时回退到 PowerShell 自带的 `Get-AuthenticodeSignature`。
- 对每个最终产物（NSIS `.exe` 与 MSI `.msi`）逐个记录 `signed` / `verified` / `signer` / `thumbprint` / `signatureStatus` 等字段，并写入 `artifacts/windows-x64/signing-manifest.json`。
- 当 `configuration.enabled = true` 但 `Get-AuthenticodeSignature` 或 `signtool verify` 报告签名无效时，发布脚本会中断并提示失败原因。

未配置任何签名相关环境变量时，`configuration.enabled = false`，`Protect-ReleaseArtifacts` 会以“未签名内部测试版”写入 manifest，不阻断内测打包流程。

## 跨平台策略校验

`scripts/verify-authenticode.mjs` 提供可在任何平台运行的策略校验：

```bash
node scripts/verify-authenticode.mjs artifacts/windows-x64/signing-manifest.json
```

支持以下策略：

- `warn-unsigned`（默认）：未签名仅发出警告，退出码 `0`，方便内测快速检查。
- `allow-unsigned`：未签名直接通过，退出码 `0`，不产生任何原因。
- `require-signed`：未签名直接失败，退出码 `1`，用于对外分发的硬门槛。
- `--thumbprint <sha1>` / `--publisher <name>`：限定允许的证书指纹或主体，多个值可重复传入。

策略与纯函数（`parseSigningManifest`、`summarizeSigningEntries`、`evaluateSigningManifest`）由 `scripts/verify-authenticode.test.mjs` 覆盖，可在不依赖 Windows 主机或真实证书的情况下与 CI 一同运行。

## 发布验收

对最终 `.exe` 和 `.msi` 逐个验证：

```powershell
Get-AuthenticodeSignature .\artifacts\windows-x64\CodeReader*.exe
Get-AuthenticodeSignature .\artifacts\windows-x64\CodeReader*.msi
signtool verify /pa /all /v .\artifacts\windows-x64\CodeReader*.exe
signtool verify /pa /all /v .\artifacts\windows-x64\CodeReader*.msi
node scripts/verify-authenticode.mjs .\artifacts\windows-x64\signing-manifest.json --require-signed
```

验收记录至少包含版本、SHA-256、签名主体、证书指纹、时间戳、验证结果和产物来源。任一产物未签名、签名无效、主体不匹配或哈希与清单不一致时，发布必须失败。

## 发布脚本集成

`scripts/release-windows.ps1` 在产物复制后调用 `scripts/sign-windows-artifacts.ps1` 的 `Protect-ReleaseArtifacts`，把签名与验证纳入发布流水线，且不要求内测阶段持有真实证书：

- 无证书时：跳过签名，仍对每个产物执行 `Get-AuthenticodeSignature`，记录为 `unsigned-internal-beta`，并写入 `artifacts/windows-x64/signing-manifest.json`，发布可继续。
- 配置证书时：用 `signtool sign`（`/fd sha256`，可配 `/tr` RFC 3161 时间戳）签名，再强制 `Get-AuthenticodeSignature` + `signtool verify /pa /all` 验证；任一产物验证失败即中断发布。
- 签名完成后重新计算 SHA-256 与体积，保证 `release-manifest.json` 与 `SHA256SUMS.txt` 与最终产物一致。

证书通过环境变量提供，不得进入仓库或日志：

| 变量                                | 用途                                                |
| ----------------------------------- | --------------------------------------------------- |
| `CODEREADER_CODESIGN_PFX_PATH`      | PFX 证书路径                                        |
| `CODEREADER_CODESIGN_PASSWORD`      | PFX 密码（仅传给 `signtool`，不写入清单）           |
| `CODEREADER_CODESIGN_SHA1`          | 证书库中证书的 SHA1 指纹（与 PFX 二选一）           |
| `CODEREADER_CODESIGN_TIMESTAMP_URL` | RFC 3161 时间戳服务地址                             |
| `CODEREADER_CODESIGN_DIGEST_ALG`    | 摘要算法，默认 `sha256`                             |
| `CODEREADER_CODESIGN_TIMESTAMP_ALG` | 时间戳摘要算法，默认 `sha256`                       |
| `CODEREADER_CODESIGN_REQUIRED`      | 设为 `1` 时无证书即拒绝发布，用于面向不受控用户分发 |

`signing-manifest.json` 记录每个产物的 `signed`、`verified`、`signatureStatus`、`signer`、`thumbprint`、`timestampSigner` 与签名配置摘要（不含密钥），作为发布验收的机器可读证据。
