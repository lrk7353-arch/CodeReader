# 内测迭代路线图

## Beta 1：工程基线

- 建立 SQLite migration runner 和旧库升级测试。
- 建立 GitHub Actions 质量门禁。
- 增加 ESLint、React Hooks 与 Prettier 门禁。
- 启用最小可用 Tauri CSP，并在实际读取阶段限制文件预览体积。
- 拆出 LLM provider 边界和前端模型工作流 hook。
- 建立内测质量、架构和审查处置文档。
- 清理异常目录与下载标记，保持仓库可复现。

## Beta 2：可诊断性与回归保护

- 后端统一 `AppError` 分类并向前端传递稳定错误码。
- 为关键工作区流程增加 React Testing Library 交互测试。
- 增加迁移失败回滚、损坏数据库和凭据不可用测试。
- 建立内测反馈模板、崩溃/错误日志脱敏规则和回归清单。
- 继续拆分工作区异步流程，为打开文件、打开项目和刷新建立可交互测试的控制器边界。
- 获得可信代码签名证书后接入 Authenticode，并把签名验证加入 Windows 发布脚本。
- 抽取用户文案资源层，为英文界面保留入口。

## Beta 3：扩展验证

- 建立 provider registry，验证第二种协议实现或 Ollama 本地实现。
- 将 prompt 版本注册、灰度和回滚纳入持久化记录。
- 在不改变 Windows 发行承诺的前提下验证 macOS/Linux 开发构建。
- 继续按职责拆分 Rust 长模块和工作区状态，不以行数作为唯一目标。

## 内测退出标准

1. 至少两个连续候选版本没有数据丢失或无法升级问题。
2. 核心阅读闭环在真实项目中完成率和失败原因可统计。
3. 严重缺陷具有自动化回归测试。
4. 安装、升级、重启恢复和模型失败降级通过验收。
5. 已知限制、隐私边界和平台支持范围对内测用户清晰可见。
