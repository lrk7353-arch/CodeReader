# CodeReader MVP 开发任务 Prompt v0.3

## 项目定位

CodeReader 是一个独立桌面端 AI 代码阅读 IDE，不是 VS Code 插件、不是聊天机器人、不是代码审查插件。

它面向使用 AI 生成代码但缺少系统编程训练的用户，核心目标是帮助用户真正阅读、理解、审阅并掌握自己的代码项目。

核心体验：

```text
打开本地项目
  -> 文件树和推荐阅读路径
  -> Monaco 展示代码
  -> 点击单行/多行/函数
  -> 右侧展示稳定解释层
  -> 用户标记已理解/有疑问/不对劲
  -> 解释保存到本地
  -> 代码变化后显示变更摘要和过期解释
```

## 技术栈

- Tauri
- React
- TypeScript
- Monaco Editor
- SQLite
- Tree-sitter
- LLM API

## 开发原则

1. 先验证阅读体验，再做完整工程。
2. 先做 Tree-sitter 结构锚点，再做行级解释生成。
3. 先做本地持久化，再做智能增强。
4. 先单文件闭环，再扩展项目级认知地图。
5. AI 生成初始解释，用户反馈和局部重新解释形成质量闭环。

## MVP 任务顺序

### Task 0：纯网页体验原型

实现 Monaco + 预置示例代码 + 预置解释 JSON。

要求：

- 支持点击单行；
- 支持多行选择；
- 支持函数级解释切换；
- 右侧解释采用三层结构；
- 支持“已理解 / 有疑问 / 不对劲”反馈。

### Task 1：Tauri + React 基础应用

创建独立桌面端应用，完成三栏布局：

```text
左侧：文件树 / 示例入口
中间：代码阅读区
右侧：解释面板
```

### Task 2：Monaco Editor 集成

要求：

- 只读展示代码；
- 显示行号；
- 支持语法高亮；
- 支持点击行；
- 支持多行选择；
- 将当前选择同步到右侧面板。

### Task 3：本地文件和项目打开

要求：

- 打开单个文件；
- 打开本地项目文件夹；
- 忽略 `node_modules`、`.git`、`dist`、`build` 等目录；
- 展示文件树；
- 自动识别 JS/TS 文件。

### Task 4：Tree-sitter 结构解析

MVP 优先支持 JavaScript / TypeScript。

要求识别：

- import/export；
- function；
- class；
- if/for/try 等代码块；
- 函数与代码块起止行；
- 行号到函数/代码块/文件的映射。

### Task 5：SQLite 和解释层 schema

至少实现：

- projects；
- files；
- code_snapshots；
- code_nodes；
- explanation_nodes；
- explanation_targets；
- explanation_feedback；
- user_reading_states；
- change_records；
- project_maps；
- reading_paths。

### Task 6：mock 解释层持久化

要求：

- 将预置解释保存到 SQLite；
- 根据文件路径、代码 hash、行/范围/结构节点加载解释；
- 关闭重开后解释仍存在；
- 用户阅读状态和反馈能恢复。

### Task 7：解释面板

解释采用三层结构：

1. 代码层意义；
2. 局部组成意义；
3. 项目全局意义。

同时展示：

- 风险；
- 前置知识；
- 审阅建议；
- 可信提示；
- 依赖行和影响行；
- 有帮助 / 不对劲 / 更白话 / 更详细 / 重新解释。

### Task 8：Context Builder

根据解释目标构造上下文：

| 目标 | 上下文 |
|---|---|
| 单行 | 当前行 + 所在函数 + 相关变量定义 + 调用处摘要 |
| 多行/代码块 | 代码块 + 所在函数 + 输入输出变量 |
| 函数 | 函数体 + 被调用函数摘要 + 调用位置 |
| 文件 | 文件结构 + import/export + 核心函数摘要 |
| 项目 | 文件树 + 入口文件 + 模块摘要 |

### Task 9：LLM 解释生成

要求：

- 输出结构化 JSON；
- 输出失败不写入数据库；
- 生成文件级、函数级、代码块级、关键行级解释；
- 支持白话模式和详细模式；
- 支持局部重新解释。

### Task 10：代码变更检测

要求：

- 计算文件 hash；
- 重新解析 AST；
- 匹配旧节点和新节点；
- 标记解释状态：
  - valid；
  - stale；
  - invalid；
  - new_unexplained；
  - deleted；
- 生成变更摘要；
- 支持只重新解释受影响区域。

### Task 11：第一公里引导

要求：

- 内置 3 个示例项目；
- 生成项目文件地图；
- 识别入口文件、配置文件、业务文件；
- 生成推荐阅读路径；
- 展示阅读进度。

### Task 12：MVP 打包

要求：

- 桌面端可运行；
- 示例体验可用；
- 单文件解释闭环可用；
- 持久化恢复可用；
- 变更摘要可用；
- 反馈和局部重新解释可用。
