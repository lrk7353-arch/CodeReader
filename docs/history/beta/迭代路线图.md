# 内测迭代路线图

## Beta 1：工程基线

- 建立 SQLite migration runner 和旧库升级测试。
- 建立 GitHub Actions 质量门禁。
- 增加 ESLint、React Hooks 与 Prettier 门禁。
- 启用最小可用 Tauri CSP，并在实际读取阶段限制文件预览体积。
- 拆出 LLM provider 边界和前端模型工作流 hook。
- 建立内测质量、架构和审查处置文档。
- 清理异常目录与下载标记，保持仓库可复现。

## Beta 4：RC 前真实验收与发布链路硬化（`0.11.0-beta.4`，收尾修复中）

Beta 4 不再继续补 MVP 能力，也不以局部功能堆叠为目标。它的定位是把 Beta 1-3 建好的工程基础拉到”可面向更广泛内测受众”的状态：真实项目能跑、失败能统计、发布链路能复验、用户体验不打断阅读，最后形成可进入 RC 候选的证据包。

- [模板完成/manual-required] 主线 1 Windows release-chain smoke：`scripts/windows-release-smoke.mjs` 自动校验 manifest/SHA-256/signing 一致性 + `Windows-release-chain-smoke.md` 人工安装清单。自动校验逻辑已修复（缺 SHA256SUMS/signing-manifest/signing entry 会判定失败）。真实安装/升级/卸载 smoke 需 Windows 主机人工执行，未完成。

## 内测退出标准

1. 至少两个连续候选版本没有数据丢失或无法升级问题。
2. 核心阅读闭环在真实项目中完成率和失败原因可统计。
3. 严重缺陷具有自动化回归测试。
4. 安装、升级、重启恢复和模型失败降级通过验收。
5. 已知限制、隐私边界和平台支持范围对内测用户清晰可见。
6. Windows 公开分发版本必须完成 Authenticode 签名，并在 CI 中通过 `node scripts/verify-authenticode.mjs --require-signed` 校验。
