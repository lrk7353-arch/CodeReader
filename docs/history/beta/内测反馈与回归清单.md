# 内测反馈与回归清单

适用阶段：`0.11.x` 内测迭代。

本文用于收集内测问题、约束日志内容，并定义每次合入前必须复核的回归路径。它不替代完整质量门禁；门禁仍以 [内测质量基线](./内测质量基线.md) 为准。

## 反馈模板

每个内测问题至少记录：

| 字段 | 要求 |
|---|---|
| 版本 | 例如 `0.11.0-beta.1`、提交号或安装包文件名 |
| 平台 | Windows 版本、是否 WSL/网络盘/外接盘项目 |
| 项目规模 | 文件数、主要语言、是否存在超大文件或二进制目录 |
| 操作路径 | 从启动到问题发生的点击顺序 |
| 输入范围 | 打开的文件、项目根目录、目标函数/行号，不粘贴敏感源码全文 |
| 期望结果 | 用户认为应该发生什么 |
| 实际结果 | UI 文案、错误码、是否丢失解释/阅读状态/模型配置 |
| 可复现性 | 必现、偶现、仅首次、仅升级后 |
| 数据影响 | 是否影响 SQLite 数据、API Key、已有解释或阅读状态 |

问题标题建议格式：

```text
[beta][模块][严重度] 简短现象
```

示例：

```text
[beta][workspace][P1] 打开包含损坏 sqlite 的用户目录后无法进入示例项目
```

## 严重度

| 等级 | 定义 | 处理要求 |
|---|---|---|
| P0 | 数据丢失、凭据泄露、应用无法启动 | 立即停止发布，先保全数据和复现材料 |
| P1 | 核心阅读闭环不可用、升级失败、模型失败覆盖已有解释 | 当前迭代必须修复并补回归测试 |
| P2 | 单一功能失败但可绕过，错误提示不清晰 | 进入最近一个 Beta 修复批次 |
| P3 | 文案、布局、低频边界体验问题 | 累积到体验迭代处理 |

## 日志与脱敏

允许记录：

- 稳定错误码，例如 `llm.timeout`、`credential.unavailable`、`db.error`。
- 文件相对路径、语言、文件大小、行数、哈希前缀。
- 操作阶段，例如 `openProject`、`refreshLoadedFile`、`generateExplanation`。
- 模型 provider 类型、模型名、端点 host。

禁止记录：

- API Key、证书私钥、PFX 密码、签名服务令牌。
- 用户源码全文、完整 prompt、完整模型响应。
- 本机用户名以外的可识别个人路径片段；需要保留路径时优先使用项目根别名和相对路径。
- 系统凭据库返回的底层敏感 payload。

脱敏规则：

1. 绝对路径只保留盘符/协议、项目根 basename 和相对路径。
2. 源码片段只保留 hash、行号范围和结构节点名。
3. 模型端点只保留 scheme、host 和 path，不保留 query/token。
4. 错误对象对外展示 `code` 和用户可执行的 `message`，不直接展示堆栈。

## 合入回归清单

每个 Beta 2 改动合入前至少检查：

- 打开示例项目后，阅读路径、文件列表、解释面板能正常显示。
- 打开单文件后，解释种子、阅读状态和工作区名称正确。
- 打开项目后，首个可预览文件能加载；不可预览文件以占位项保留在树中。
- 刷新未变化文件不会生成新的解释；刷新变化文件会保留可迁移解释并标记受影响解释。
- 无 API Key 时远程模型生成返回 `credential.not_set`，不会覆盖已有解释。
- 凭据库不可用时返回 `credential.unavailable`，不会清空已保存模型配置。
- 损坏数据库打开失败时不覆盖原文件。
- 迁移失败时 `user_version` 不前进，旧数据不被半迁移状态污染。
- 顶栏阶段身份与版本号一致显示 `内测 · Beta 2`，`package.json` / `src-tauri/tauri.conf.json` 同步为 `0.11.0-beta.2`。
- 在未配置签名证书时，`npm run release:windows` 能完成内测打包，并在 `artifacts/windows-x64/signing-manifest.json` 中记录为 “未签名内部测试版”。
- `node scripts/verify-authenticode.mjs` 在不依赖 Windows 主机的 CI 环境下通过，签名策略变更时同步更新 `scripts/verify-authenticode.test.mjs`。

## 自动化门禁

合入内测主线前执行：

```bash
npm test
npm run lint
npm run format:check
npm run build
npm run cargo:test
npm run cargo:clippy
npm run cargo:check
git diff --check
```

若只完成局部开发，可以先跑相关测试，但最终合入仍必须跑完整门禁。
