# 架构入口

## 1. 运行结构

```text
React UI
  -> desktopWorkspace.ts (Tauri command adapter)
  -> Rust command modules
  -> SQLite / Tree-sitter / LLM provider
```

浏览器模式只用于无 API Key 示例预览；真实文件、SQLite、系统凭据和模型调用运行在 Tauri 桌面端。

## 2. 前端入口

| 文件 | 职责 |
|---|---|
| `src/app/App.tsx` | 工作区容器、文件/项目选择、阅读状态协调 |
| `src/app/hooks/useExplanationContext.ts` | 当前解释的 Context Bundle 生命周期 |
| `src/app/hooks/useModelWorkflow.ts` | 模型配置、发送确认、生成状态和结果回写 |
| `src/services/desktopWorkspace.ts` | 前端到 Tauri command 的唯一适配层 |
| `src/features/file-explorer/` | 项目树、解释目标和阅读路径导航 |
| `src/features/explanation-panel/` | 解释、风险、上下文和反馈展示 |

## 3. Rust 入口

| 模块 | 职责 |
|---|---|
| `code_service.rs` | 文件读取、项目扫描、语言和预览能力 |
| `context_builder.rs` | 结构分析、上下文预算和来源证明 |
| `explanation_service.rs` | prompt、结构校验、解释持久化编排 |
| `llm_provider.rs` | LLM provider trait 与 OpenAI-compatible HTTP 实现 |
| `persistence_service.rs` | 解释、阅读状态、变更记录和模型配置仓储 |
| `persistence/schema.rs` | SQLite schema 与顺序迁移 |
| `change_detection.rs` | 快照差异、解释迁移和过期判定 |
| `project_guidance.rs` | 项目地图、推荐阅读路径和进度 |

## 4. 数据库迁移

`open_database` 每次打开数据库时调用 `persistence::schema::migrate`。迁移读取 `PRAGMA user_version`，逐个执行缺失版本，并在成功后更新游标。任何 schema 修改必须新增迁移版本，不能只修改初始建表 SQL。

## 5. 解释生成链路

```text
用户选择目标
  -> Context Builder 生成有限上下文
  -> 用户确认发送
  -> LlmProvider 完成协议调用
  -> 结构解析与语义校验
  -> 必要时进行一次结构修复
  -> SQLite 保存新解释
```

模型调用失败或结构连续无效时，已有解释保持不变。
