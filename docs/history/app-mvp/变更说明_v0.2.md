# CodeReader 文档变更说明 v0.2

## 1. 变更原因

上一版文档中，MVP 被建议为：

```text
VS Code Extension + SQLite + Tree-sitter/LSP + LLM API
```

这个方案虽然有工程验证效率，但会导致产品定位偏离 CodeReader 的核心愿景。

CodeReader 的目标不是成为 VS Code 的附属插件，而是成为一个独立的、围绕 AI 代码阅读和代码认知掌握而设计的新型 IDE 式产品。

因此，本次文档将 MVP 主线修正为：

```text
Tauri + React + Monaco Editor + SQLite + Tree-sitter + LLM API
```

## 2. 关键变更

| 事项 | 旧版本 | v0.2 新版本 |
|---|---|---|
| MVP 形态 | VS Code 插件优先 | 独立桌面端应用优先 |
| 产品定位 | 类 IDE 插件 | 独立 AI 代码阅读 IDE |
| 代码展示 | 依赖 VS Code 编辑器 | 使用 Monaco Editor 构建自有阅读区 |
| 数据存储 | 插件本地存储/SQLite | 应用级 SQLite 本地解释层 |
| 插件角色 | MVP 主体 | 后续补充入口/分发渠道 |
| 核心体验 | 在 VS Code 里解释代码 | 在 CodeReader 中系统阅读代码 |

## 3. 新定位

CodeReader v0.2 文档统一采用以下定位：

> CodeReader 是一个独立的、IDE 形态的 AI 代码阅读器，通过持久化的行级解释层，帮助 AI 编程用户逐行、逐块、逐函数、逐文件理解并掌握自己的代码项目。

## 4. VS Code 插件的新角色

VS Code 插件不再作为 MVP 主体。

它的新角色是：

1. 后续生态入口；
2. 从 VS Code 一键打开 CodeReader 的桥接工具；
3. 展示 CodeReader 解释状态的轻量侧边能力；
4. 面向重度开发者的补充分发渠道。
